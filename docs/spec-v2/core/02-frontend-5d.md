# Frontend 5D Definition

## Purpose

Freeze the five dimensions required for every frontend page task.

## D1 Style

Checks shell, layout, visual hierarchy, panel presence, card or table choice,
theme consistency, and the location of important controls.

## D2 Click Event

Checks that every primary action triggers the correct UI transition.

## D3 Field Mapping

Checks that API fields land in the correct UI slot without fake placeholders.

## D4 State Mapping

Checks state transitions and state-specific UI.

## D5 Quantity And Constraints

Checks count-based and structure-based constraints.

## Page Completion Rule

A page is not complete unless all 5 dimensions have at least one executable
assertion path.
