# Packet Template

Use this structure for any new packet added later.

## Header

- packet id
- scope
- owner type: frontend, api, backend, or mixed

## Inputs

- source files
- dependent packets

## Task Breakdown

For pages:

1. D1 style
2. D2 click event
3. D3 field mapping
4. D4 state mapping
5. D5 quantity and constraints

For APIs:

1. request contract
2. response contract
3. error contract
4. contract tests
5. integration tests

For backend packets:

1. logic tree
2. test tree
3. dependent pages
4. dependent APIs

## Output

- implementation files
- tests
- verification commands
- handoff notes
